Sentinel

RFID Attendance Tracking for HMCS Chippawa

What is Sentinel?

Sentinel is a standalone attendance tracking system built for HMCS Chippawa, a Royal Canadian Navy reserve unit in Winnipeg. The system uses RFID badge scanning for fast, contactless check-ins and operates independently from DWAN, with full offline support for reliable operation.

Key Value Proposition: Simple, reliable attendance tracking that works offline, with easy Excel-based member import and admin-friendly interfaces designed for non-technical users.

Kiosk Terminals

Raspberry Pi 5 devices with NFC readers for badge scanning. Supports offline queuing when disconnected.

Dashboard

Real-time web interface for monitoring presence, managing employees, and viewing reports.

Backend API

RESTful API with WebSocket support for real-time updates across all connected clients.

Core Features

1. RFID Badge Scanning

  • Contactless check-in/out - Members tap their badge on the NFC reader
  • Sub-second recognition - Badge ID lookup and confirmation in under 500ms
  • Visual and audio feedback - Clear success/error states with sound
  • Automatic direction detection - System determines if it's a check-in or check-out based on current state

2. Offline-First Architecture

  • Local queue storage - Check-ins are stored locally when network is unavailable
  • Automatic sync - Queued events synchronize when connection is restored
  • Conflict resolution - Server-side timestamp reconciliation for accurate records
  • Status indicators - Clear visual feedback showing online/offline/syncing states

3. Real-Time Presence Monitoring

  • Live dashboard updates - WebSocket-powered instant updates across all clients
  • TV display mode - Large-format 10-foot UI for wall-mounted displays
  • At-a-glance metrics - Present count, absent count, late arrivals, visitors
  • Activity feed - Scrolling list of recent check-ins

4. Visitor Management

  • Self-service sign-in - Touch-screen form for visitors without badges
  • Host notification - Alert the employee being visited
  • Visitor badges - Temporary badge assignment for recurring visitors
  • Sign-out tracking - Complete visitor audit trail

5. Administrative Controls

  • Member management - Add, edit, deactivate members and badges
  • Division organization - Group members by division/department
  • Manual adjustments - Admin override for missed punches or corrections
  • Reporting - Attendance reports, parade state summaries

6. Excel Nominal Roll Import

Designed for non-technical administrators. Upload the unit's nominal roll spreadsheet and let the system handle the rest.

  • Drag-and-drop upload - Simple file selection, no complex forms
  • Smart column detection - Automatically identifies name, service number, rank, division columns
  • Error handling with guidance - Clear, plain-language explanations of any issues found
  • Preview before commit - Review all changes (new members, updates, potential duplicates) before saving
  • Fuzzy matching - Handles typos, name variations, and format inconsistencies
  • Change history - Full audit trail of what was imported and when
Admin-Friendly Design: Every screen includes help text. Error messages explain what went wrong AND how to fix it. No technical jargon. Large buttons, clear labels, confirmation dialogs for destructive actions.

7. Events & Temporary Groups

Host special events with temporary building access for non-unit personnel. Perfect for court martials, training courses, inspections, and ceremonies where external attendees need temporary NFC badges.

  • Event creation - Define event name, dates, and access window
  • Attendee management - Add participants manually or via Excel/CSV import
  • Temporary badge pool - Reusable visitor badges (V-###) assigned to event attendees
  • Automatic expiration - Badges automatically disabled when event ends
  • Separate monitoring - Dedicated views for event attendance, distinct from unit members
  • Badge recovery tracking - Track which badges have been returned after event ends
  • Audit trail - Complete log of all event badge scans for security
Member Classifications:
  • Full-Time Staff - Class B/C personnel, present daily
  • Reserve Members - Class A personnel, parade nights + exercises
  • Event Attendees - Temporary access tied to specific events

User Flows

Member Check-In

1

Approach Kiosk

Member sees "Tap Your Badge" prompt with pulsing animation

2

Tap Badge

NFC reader detects badge and reads unique ID

3

Lookup & Record

System matches badge ID to member, records timestamp

4

Confirmation

Success screen shows member name, rank, action (In/Out), and time. Returns to idle after 3 seconds.

Visitor Sign-In

1

Select Visit Type

Choose reason: Meeting Someone, Work/Contractor, Recruitment, Course/Training, Special Event, or Other

2

Link to Event (if applicable)

If "Special Event" selected, choose from active events (Court Martial, Training Course, etc.). Pre-registered attendees can scan their badge.

3

Enter Details

Name, organization, host member (searchable list). Additional context based on visit type.

4

Confirmation

Visitor receives confirmation. Host notified. Visit logged with type/event for reporting.

Visit Types: General Visit, Contractor/Work, Recruitment, Course/Training, Event Attendee, Official Visit. Admin can customize these categories and run reports by type (e.g., track recruitment visits over time).

Offline Recovery

1

Network Lost

Kiosk detects disconnection, shows "Offline" indicator

2

Continue Operating

Check-ins continue normally, stored in local queue with timestamps

3

Connection Restored

Kiosk detects network, shows "Syncing" with queue count

4

Sync Complete

All queued events uploaded, dashboard updates with historical data

Excel Nominal Roll Import

1

Upload File

Admin drags Excel file onto upload area or clicks to browse. Accepts .xlsx and .xls

2

Column Mapping

System auto-detects columns. Admin confirms or adjusts if needed. Shows sample data for verification.

3

Review Changes

Preview shows: new members to add, existing members to update, potential issues (duplicates, missing data). Each issue has a "How to fix" suggestion.

4

Confirm Import

Admin clicks "Import" to apply changes. Success summary shows what was added/updated. Option to undo within 24 hours.

Event Setup (e.g., Court Martial)

1

Create Event

Admin enters event name (e.g., "Court Martial - R v. Smith"), code (CM-2025-001), dates, and description.

2

Add Attendees

Import from Excel/CSV (downloaded from DWAN) or add manually. Include name, rank, organization, and role.

3

Assign Badges

Scan visitor badges to assign to attendees, or use bulk assignment. Each badge is linked to the event.

4

Monitor & Close

Track attendance via dedicated event view. When event ends, badges auto-expire. Collect badges using recovery checklist.

Technical Architecture

Hardware Deployment

Device Hardware Location
Backend Server Raspberry Pi 5 (8GB RAM, 128GB storage) Server room / secure location
Primary Entrance Kiosk Raspberry Pi 5 + Waveshare PN532 NFC HAT + GeeekPi 10.1" Touchscreen Main entrance / quarterdeck
Rear Door Scanner Raspberry Pi 5 + Waveshare PN532 NFC HAT + Speaker (no display) Rear entrance
Standalone System: All devices communicate over local network only. No DWAN connection required. The system is completely self-contained within the unit.

Software Stack

TypeScript React Node.js PostgreSQL Redis WebSocket REST API

Data Flow

Kiosk → NFC scan event → Local Queue (if offline) → Backend API → Database → WebSocket broadcast → Dashboard/TV

Key Technical Requirements

Requirement Target
Scan-to-confirmation latency < 500ms (online), < 200ms (offline)
Offline queue capacity 10,000+ events per kiosk
Dashboard update latency < 1 second via WebSocket
Concurrent kiosks supported 100+ per server instance
Uptime target 99.9% (kiosks operate offline if server down)

API Overview

REST Endpoints

Endpoint Method Description
/api/checkin POST Record a badge scan event
/api/checkin/bulk POST Sync offline queue (batch upload)
/api/members GET, POST List/create members
/api/members/:id GET, PUT, DELETE Manage individual member
/api/presence GET Current presence status for all members
/api/visitors GET, POST Visitor sign-in/out management
/api/reports/attendance GET Attendance reports with date filters
/api/import/nominal-roll POST Upload Excel file for member import
/api/import/preview POST Preview import changes before committing

WebSocket Events

Event Direction Description
checkin Server → Client New check-in recorded
presence_update Server → Client Member presence status changed
visitor_signin Server → Client New visitor signed in
kiosk_status Client → Server Kiosk heartbeat/health report
