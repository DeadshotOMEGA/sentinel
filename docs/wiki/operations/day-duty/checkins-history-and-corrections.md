# Check-ins History and Corrections

> Audience: Duty Day Staff (DDS), managers, admins

## Purpose

Use this page to review check-in events and apply controlled corrections when verified operationally necessary.

## Working Rules

- Treat historical corrections as auditable actions.
- Filter scope before editing (date, direction, division).
- Record why any correction was made.

## Procedure

1. Filter to the smallest relevant scope.
2. Locate target entry and verify source evidence.
3. Apply correction fields carefully.
4. Save and confirm resulting record state.
5. Notify watch leadership for high-impact corrections.
