# DDS Daily Workflow

> Audience: Duty Day Staff (DDS), duty supervisors, operations admins

[Open Sentinel Dashboard](http://sentinel.local/dashboard)

## Purpose

This procedure defines the recommended operational sequence for Duty Day Staff (DDS).

## Preconditions

- You are signed in with an authorized operational account.
- You have confirmed your assigned role for this watch period.
- You can reach `sentinel.local` on the operations network.

## Recommended Sequence

1. Open Dashboard and triage Security Alerts first.
2. Verify status blocks (DDS, Duty Watch, Building, Lockup Holder).
3. Execute validated Quick Actions only.
4. Confirm outcome in Presence Grid and check-ins records.
5. Escalate mismatched state before continuing high-impact actions.

## Validation Checks

- Dashboard values match current operational reality.
- Actions taken are reflected in check-ins and presence data.
- Unclear or conflicting state is escalated with evidence.

## Failure / Escalation

- If alerts or status data are stale, refresh once and revalidate.
- If lockup/building controls diverge from expected state, stop and escalate.
- Capture timestamp and screenshot evidence for incident handoff.
