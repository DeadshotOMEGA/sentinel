# Feature: Events & Temporary Groups

## Problem Statement

HMCS Chippawa occasionally hosts special events requiring temporary building access for non-unit personnel. Examples:
- **Court Martial** - High-ranking officers, judges, lawyers, witnesses (~1 month)
- **Training courses** - Visiting instructors and students
- **Inspections** - External assessors
- **Ceremonies** - Dignitaries, guests

These attendees need:
1. Temporary NFC badge assignment
2. Tracking separate from regular unit members
3. Dedicated monitoring views
4. Automatic access expiration
5. Audit trail for security

---

## Member Classification

### Existing: Unit Members
| Type | Description |
|------|-------------|
| **Full-Time Staff** | Class B/C personnel, present daily |
| **Reserve Members** | Class A personnel, parade nights + exercises |

### New: Event Attendees
| Type | Description |
|------|-------------|
| **Event Participant** | Temporary access tied to specific event |

---

## Data Model

### Event
```typescript
interface Event {
  id: string;
  name: string;                    // "Court Martial - R v. Smith"
  code: string;                    // "CM-2025-001" (short identifier)
  description: string;

  // Timeline
  startDate: Date;
  endDate: Date;

  // Access control
  status: "draft" | "active" | "completed" | "cancelled";
  autoExpireBadges: boolean;       // Disable badges on endDate

  // Metadata
  createdBy: string;
  createdAt: Date;
  updatedAt: Date;
}
```

### Event Attendee
```typescript
interface EventAttendee {
  id: string;
  eventId: string;

  // Identity
  name: string;
  rank?: string;                   // Military rank if applicable
  organization: string;            // "JAG", "Base Esquimalt", etc.
  role: string;                    // "Judge", "Witness", "Counsel"

  // Badge assignment
  badgeId?: string;                // Assigned NFC badge
  badgeAssignedAt?: Date;

  // Access window (can override event dates)
  accessStart?: Date;
  accessEnd?: Date;

  // Status
  status: "pending" | "active" | "checked-out" | "expired";

  // Audit
  createdAt: Date;
  updatedAt: Date;
}
```

### Badge Pool
```typescript
interface Badge {
  id: string;
  serialNumber: string;            // NFC UID

  // Assignment
  assignmentType: "member" | "event" | "unassigned";
  assignedToId?: string;           // Member ID or EventAttendee ID

  // Tracking
  lastUsed?: Date;
  status: "active" | "disabled" | "lost" | "returned";
}
```
